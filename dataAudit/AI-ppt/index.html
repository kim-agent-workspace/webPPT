Problem · 传统随访

三大核心问题

问题一

标准化模板

"高危每月一次、低危每三个月一次" -- 分级依据人群统计规则，而非患者当前具体病情。

问题二

事后模式

患者病情进展已经发生，才能在下一次定期随访中被发现和响应 -- 此时干预窗口已经错过。

问题三

定期不等于及时

真正有价值的是在进展发生之前就调整管理强度，而非等待下一次固定随访日期。

一个血糖控制良好但血脂恶化的患者和一个肾功能已下降的患者，可能在模板中同属一个等级、套用同一套随访方案。
传统随访 · 标准化模板 + 事后模式 + 定期不等于及时

Problem · 传统处方的局限

三个问题

问题一
个体针对性弱
统一模板对任何具体患者都不够精准。每个患者病情组合、风险因素、生活方式各不相同，模板化处方无法覆盖个体差异。
问题二
患者执行难
"每日蛋白质60克" -- 患者无法转化为日常行为。"低盐饮食"、"适量运动"这类建议缺乏可操作的量化标准。
问题三
指导不接地气
没有结合当地饮食文化、季节气候特征。同样的建议在北方和南方、夏季和冬季的执行效果截然不同。
传统处方的局限 · 个体针对性弱 + 患者执行难 + 指导不接地气

Solution · 个体化与可操作

个体化与可操作

2.1 解决个体针对性弱

基于患者全部慢病和实际异常指标驱动处方内容生成。eGFR下降则肾脏危险信号排首位；尿酸偏高则饮水和低嘌呤内容强化。

一份处方体现的是这个患者此刻的综合风险画像。

2.2 解决患者执行难

建议全部翻译为患者可直接操作的具体行为。

蛋白质："1个鸡蛋 + 1杯牛奶 + 2.5两瘦肉 + 1块豆腐"
运动强度："能说话不能唱歌 = 刚好"
盐摄入："盐不超过5克/天，约等于1啤酒瓶盖"

日常监测计划 + 环境适应指导 + 随访安排，帮助患者在院外执行管理方案。
Part 06
短信浓缩版
数千字完整版压缩为核心要点，短信推送到患者手机，确保关键信息触达。

差异化总结

差异一

从"管一件事"到"管一个人"

传统HIS以科室为边界，本系统以患者为中心对其全部慢病统一管理

差异二

从"事后记录"到"事前干预"

AI预测模型在指标恶化之前即识别出风险上升趋势

差异三

从"被动工具"到"路径引导"

每个环节给出结构化建议并提示下一步，弥合不同年资医生的能力差异
